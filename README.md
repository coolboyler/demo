# 负荷预测展示 Demo

一个基于 `FastAPI + HTML + 原生 JS` 的可公网访问 demo，用于展示未来 24 小时负荷预测、电量趋势、历史准确率，并支持：

- 页面按日期切换查看指定日的负荷预测与负荷实际
- 同时展示按日准确率和按月准确率
- 外部系统通过接口推送 24 小时预测值
- 外部系统通过一个接口同时上传同一目标日的负荷预测与负荷实际
- 页面手动录入预测值，支持从 Excel 直接粘贴 24 个值并自动识别
- 页面或接口回填历史负荷实际，支持整行/整列粘贴
- 自动计算历史准确率 / MAE / RMSE / 5% 命中率

默认数据持久化在 `data/store.sqlite3`。
如果历史上存在 `data/store.json`，应用首次启动时会自动迁移到 SQLite，之后以 SQLite 为准。

## 快速启动

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python run.py
```

启动后访问：

- 页面：`http://127.0.0.1:8000/`
- 健康检查：`http://127.0.0.1:8000/healthz`

## 公网访问

开发机或服务器上直接监听 `0.0.0.0`：

```bash
python run.py
```

如果部署到云服务器，放行 `8000` 端口即可直接访问；正式环境建议再加一层 Nginx 反代到 `80/443`，并补上 HTTPS 与鉴权。

## 主要接口

### 1. 联合上传目标日负荷预测与负荷实际

`POST /api/load-records`

```json
{
  "target_date": "2026-03-06",
  "forecast": {
    "source": "dispatch-center",
    "site_name": "华海升园区",
    "note": "联合上传",
    "values": [720, 715, 702, 690, 682, 688, 710, 760, 845, 900, 936, 952, 968, 974, 960, 942, 926, 915, 896, 860, 820, 790, 765, 742]
  },
  "actual": {
    "source": "ems",
    "values": [710, 705, 694, 683, 676, 682, 701, 752, 832, 889, 922, 940, 957, 963, 949, 935, 920, 906, 887, 851, 812, 782, 758, 734]
  }
}
```

### 2. 单独写入预测

`POST /api/forecasts`

```json
{
  "target_date": "2026-03-07",
  "source": "dispatch-center",
  "site_name": "华海升园区",
  "note": "外部系统推送",
  "values": [720, 715, 702, 690, 682, 688, 710, 760, 845, 900, 936, 952, 968, 974, 960, 942, 926, 915, 896, 860, 820, 790, 765, 742]
}
```

### 3. 单独回填实绩

`POST /api/actuals`

```json
{
  "target_date": "2026-03-06",
  "source": "ems",
  "values": [710, 705, 694, 683, 676, 682, 701, 752, 832, 889, 922, 940, 957, 963, 949, 935, 920, 906, 887, 851, 812, 782, 758, 734]
}
```

### 4. 查询仪表盘

`GET /api/dashboard`

返回页面所需的总览、最新预测、历史准确率和接口示例数据。

### 5. 查询按日记录

`GET /api/history?limit=30`

返回按目标日聚合后的记录列表，若同一天有多次预测，仅统计最新一条用于准确率计算。

## 可选鉴权

如果设置了环境变量 `DEMO_API_KEY`，则写接口需要携带请求头：

```bash
X-API-Key: your-token
```

读取接口和首页默认开放，便于做演示。
