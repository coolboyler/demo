# 负荷预测算法文档

## 概述

本文档描述一套用于电力负荷预测的**规则+分流模型**，核心思想是：
- **基线规则**：基于历史数据的加权融合
- **分流机制**：根据日期类型路由到不同的专用预测策略
- **无机器学习**：全程参数网格搜索，无需 GPU，可解释性强

---

## 一、基线规则 (D-N 规则)

### 1.1 问题定义

**发单口径预测**：在目标日 D 的前 N 天（如 D-5）发单，需要预测 D 日的 24 小时负荷曲线。

**约束条件**：
- 只能使用 `issue_date` 及之前的历史数据
- 不能使用未来信息（天气等）

### 1.2 基线预测公式

```
基线预测 = α_short × 近期加权 + α_same × 同类型缩放 + α_weekly × 周周期加权
```

其中：
```
α_short + α_same + α_weekly = 1
```

### 1.3 三路融合详解

#### (1) 近期加权

使用 D-N 到 D-N-6 共 7 天的负荷曲线，指数衰减加权：

```
近期加权[h] = Σ (w_i × lag_{N+i}_h) / Σ w_i
```

```
w_i = exp(-λ_short × i),  i = 0, 1, ..., 6
```

| 参数 | 含义 |
|-----|------|
| λ_short | 近期权重衰减率，值越大越看重最近几天 |
| N | 发单提前天数（如 N=5） |

#### (2) 同类型缩放

从历史中找最近的 1-2 个**同类型日期**（如同是工作日、同是春节），取其负荷曲线并缩放：

```
同类型曲线[h] = mean(历史同类型日最近1-2天的 h 点负荷)
```

```
缩放因子 = 近期加权的日总量 / 同类型曲线的日总量
```

```
同类型缩放[h] = 同类型曲线[h] × 缩放因子
```

#### (3) 周周期加权

使用 D-7, D-14, D-21 共 3 周的同星期负荷曲线，指数衰减加权：

```
周周期加权[h] = Σ (w_j × lag_{7×j}_h) / Σ w_j
```

```
w_j = exp(-λ_weekly × j),  j = 1, 2, 3
```

### 1.4 参数训练

通过网格搜索在验证集上寻找最优参数组合：

| 参数 | 搜索范围 | 步长 |
|-----|---------|------|
| λ_short (short_lambda) | [0.05, 0.60] | ~0.025 |
| α_same (same_alpha) | [0.00, 0.50] | 0.05 |
| λ_weekly (weekly_lambda) | [0.05, 0.60] | ~0.05 |
| α_weekly (weekly_alpha) | [0.00, 0.50] | 0.05 |

**约束**：`α_same + α_weekly ≤ 0.70`

**目标函数**：验证集 WAPE 最小化

```
WAPE = Σ|实际值 - 预测值| / Σ|实际值| × 100%
```

### 1.5 训练产物

一个 JSON 文件，包含 4 个浮点数：
```json
{
  "short_lambda": 0.25,
  "same_alpha": 0.15,
  "weekly_lambda": 0.10,
  "weekly_alpha": 0.20
}
```

## 二、最优模型 (假期分流模型)

### 2.1 模型架构

```
┌─────────────────────────────────────────────────────────┐
│                     输入：目标日 D                        │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
                ┌───────────────────────┐
                │     日期类型判断       │
                └───────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼
   ┌─────────┐        ┌──────────┐        ┌─────────┐
   │节假日相关│        │ 调休工作日│        │  普通日 │
   └─────────┘        └──────────┘        └─────────┘
        │                   │                   │
        ▼                   ▼                   ▼
   节假日专模          调休专模            相似日融合
        │                   │                   │
        └───────────────────┼───────────────────┘
                            │
                            ▼
                ┌───────────────────────┐
                │     是否激活判断       │
                └───────────────────────┘
                            │
                            ▼
                ┌───────────────────────┐
                │    最终预测输出        │
                └───────────────────────┘
```

### 2.2 日期分类

根据 `target_refined_date_type` 字段分类：

| 分类 | 示例 | 说明 |
|-----|------|------|
| **holiday** | `holiday:Spring Festival_d1` | 节假日当天 |
| **pre** | `pre_Spring Festival_d3` | 节前 N 天 |
| **post** | `post_National Day_d2` | 节后 N 天 |
| **makeup** | `makeup_workday_w1` | 调休工作日 |
| **ordinary** | `workday_w1`, `weekend_w2` | 普通工作日/周末 |

### 2.3 节假日专模

#### 适用日期
- 节假日当天 (holiday)
- 节前 N 天 (pre)
- 节后 N 天 (post)

#### 算法

**节假日当天**：
```
1. 找到同节日家族的历史数据（如历年春节）
2. 按节日位置做线性插值
3. 取多年均值
```

**插值公式**：
```python
# 设目标日在节日中位置为 pos，节日总长度为 len
target_x = (pos - 1) / (len - 1)  # 归一化到 [0, 1]

# 对每个小时 h，在历史节日曲线上插值
pred[h] = np.interp(target_x, source_x, source_loads[h])
```

**节前/节后肩部日**：
```
1. 找到历史相同 key 的日期（如 pre_Spring Festival_d3）
2. 取最近 1-2 年的均值
```

### 2.4 调休专模

#### 适用日期
- 纯调休工作日（`makeup_workday_w*`，非春节前后）

#### 算法

```
预测日总量 = 锚点日总量 × 调休倍率中位数
预测曲线 = 预测日总量 × 历史调休日归一化曲线均值
```

**步骤**：

1. **计算锚点日总量**：预测日之前最近 3 个正常工作日的平均日负荷
2. **计算调休倍率**：历史调休日日总量 / 该调休日之前的锚点日总量
3. **预测日总量** = 锚点日总量 × median(历史调休倍率)
4. **预测曲线** = 预测日总量 × mean(历史调休日归一化曲线)

### 2.5 普通日相似日融合

#### 适用日期
- 普通工作日 (workday_w*)
- 普通周末 (weekend_w*)

#### 相似度打分

对每个历史候选日计算相似度得分：

```
得分 = Σ 打分项
```

| 打分项 | 条件 | 得分 |
|-------|------|------|
| 相同日期类型 | `target_refined_date_type` 相同 | +10 |
| 相同节日上下文标签 | `holiday_context_tag` 相同 | +9 |
| 相同节日家族 | `holiday_context_family` 相同 | +4 |
| 相同上下文侧边 | `holiday_context_side` 相同 | +2 |
| 相同上下文桶 | `holiday_context_bucket` 相同 | +2.5 |
| 相同星期 | `target_dayofweek` 相同 | +4 |
| 月份接近度 | `target_month` 接近 | 0~4 (距离越小越高) |
| 日期接近度 | `target_dayofmonth` 接近 | 0~3 |
| 年份锚点距离 | 与去年同日期距离 | 0~6 |
| 星期锚点距离 | 与去年同星期距离 | 0~5 |
| 近期优先 | 与发单日距离 | 0~1.5 |

#### 节日上下文标签

对于非节假日的普通日，根据其与最近节假日的距离生成上下文标签：

```
若 距上个节假日 ≤ 28天 且 ≤ 距下个节假日:
    标签 = "after:{节日家族}:w{第几周}"

若 距下个节假日 ≤ 28天:
    标签 = "before:{节日家族}:w{第几周}"

否则:
    标签 = "ordinary:{日期类型组}"
```

#### 预测融合

```
1. 从历史中选出得分最高的 Top-K 个候选日
2. 按得分归一化得到权重
3. 加权平均得到相似日预测曲线
4. 与基线预测做线性融合
```

```
最终预测 = (1 - α) × 基线预测 + α × 相似日预测
```

#### 参数搜索

| 参数 | 搜索范围 |
|-----|---------|
| K (top_k) | {3, 5, 7} |
| α (alpha) | {0.2, 0.35, 0.5, 0.65} |

## 三、激活机制

### 3.1 节日家族激活

对每个节日家族（如春节、国庆），在训练集上回测：

```
回测条件：
1. 使用 N-1 年的历史数据预测 N 年的节日相关日
2. 计算专用预测的准确率
3. 计算基线预测的准确率
4. 计算提升幅度 = 专用准确率 - 基线准确率
```

**激活条件**：
- 样本数 ≥ 2
- 准确率提升 > 0.5%

### 3.2 普通日激活

对 workday 和 weekend 分别回测：

**激活条件**：
- 样本数 ≥ 20
- 准确率提升 > 0.2%

### 3.3 调休激活

**激活条件**：
- 样本数 ≥ 1
- 准确率提升 > 0%

## 四、预测流程

```python
def predict(target_date, history, config):
    # 1. 判断日期类型
    date_type = classify_date(target_date)

    # 2. 获取基线预测
    baseline = rule_prediction(target_date, history, rule_params)

    # 3. 根据类型路由
    if date_type == "holiday" and family_active:
        pred = holiday_special_profile(target_date, history)
    elif date_type == "makeup" and makeup_active:
        pred = makeup_special_profile(target_date, history)
    elif date_type == "ordinary" and ordinary_active:
        similar = similar_day_profile(target_date, history, top_k)
        pred = (1 - alpha) * baseline + alpha * similar
    else:
        pred = baseline

    return pred
```

## 五、评估指标

### 5.1 日总量准确率

```
准确率 = 100% - WAPE
```

```
WAPE = Σ|实际日总量 - 预测日总量| / Σ|实际日总量| × 100%
```

### 5.2 小时级准确率

```
WAPE_hourly = Σ|实际值 - 预测值| / Σ|实际值| × 100%
```

### 5.3 偏差率

```
Bias = (预测日总量 - 实际日总量) / 实际日总量 × 100%
```

## 七、算法特点

### 优点
1. **可解释性强**：每一步都能追溯原因
2. **资源消耗低**：无需 GPU，无需模型文件
3. **冷启动友好**：新站点只需历史数据即可
4. **节日表现好**：专用策略处理节日
5. **易于调试**：预测出错可快速定位

### 局限性
1. **依赖历史相似性**：极端天气、突发事件处理不佳
2. **无天气特征**：未引入温度等气象信息
3. **参数固定**：不像 ML 模型能自动学习复杂模式
